## Inbox Tabs

GitDesk organizes your PRs into two focused views:

<CardGrid>
  <Card title="Review Requests" icon="approve-check-circle">
    PRs waiting for your review, plus PRs you've recently reviewed. Your review queue at a glance.
  </Card>
  <Card title="Created" icon="pencil">

  </Card>

### Review Requests Tab

This tab shows PRs that need your attention:

<Aside type="note" title="Pending Reviews">
PRs where you've been requested as a reviewer. Each card shows the PR title, repository, author, and time since last update.
</Aside>

<Aside type="tip" title="Recently Reviewed">

### Created Tab

Your authored PRs organized by status:

<Aside type="note" title="Open Pull Requests">

</Aside>

<Aside type="tip" title="Recently Created">
Quick access to your most recently created PRs, regardless of current status.

## PR Card Anatomy

Each PR card displays essential information at a glance:

| Element | Description |
|---------|-------------|
| **Title** | The pull request title ‚Äî click to open on GitHub |
| **Repository** | Full `owner/repo` format with link |
| **Author** | PR author with GitHub avatar |
| **Labels** | Color-coded PR labels |
| **Branches** | Base ‚Üê Head branch information |
| **Updated** | Relative time since last activity |

## Quick Actions

| Action | How |
|--------|-----|
| **Open PR** | Click any PR card to open it on GitHub |
| **Search** | Use the search bar to filter by title, repo, or author |
| **Refresh** | Click the refresh button for immediate update |
| **Theme** | Toggle dark/light mode in the header |

## Background Refresh

GitDesk keeps your inbox current automatically:

- **Auto-refresh** every 5 minutes while running
- **Cached data** for instant loads on startup
- **Manual refresh** available anytime via header button
